# Game Art Direction

Use this skill before generating or integrating any player-facing art.

## Workflow

1. Confirm the asset belongs to the original RO-like 2D/2.5D direction.
2. Define the visual role: character, equipment, monster, scene, UI, item, skill, pack, market, or challenge.
3. Check boundaries:
   - original only; do not copy RO assets, logos, UI, or protected designs.
   - bright fantasy MMORPG tone, readable mobile silhouettes, clean outlines.
   - no CSS, HTML, or web-card art for the player game screen.
4. Produce or review against a style bible entry before integration.

## Required Output

- asset category
- visual intent
- palette and line direction
- forbidden references or copy risks
- acceptance note for Godot integration
