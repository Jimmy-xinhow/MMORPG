# 幸運禮包 MMORPG MVP 產品說明

## 這套系統在做什麼

這是一個以 MMORPG 遊戲內容為核心的幸運禮包與營運收入管理原型。

玩家流程是：

1. 玩家持有或購買遊戲內幸運禮包。
2. 禮包可以在交易場中被掛單與購買。
3. 禮包開啟後產出遊戲內容，例如道具或素材。
4. 開啟後的禮包會銷毀，避免同一包被重複交易或重複開啟。

經營者收入流程是：

1. 平台建立 `OperatorSettlement`，代表已核准的經營收入。
2. 系統會區分總額、可提現核准額、排除額與排除理由。
3. 只有核准後的可提現金額可以申請撥款。
4. 撥款必須經過核准、稅務完成、付款資料通過、排程付款、標記已付款。

合規邊界是：

1. GC、禮包、道具、票券、Boss 獎勵、公會獎勵、玩家交易價差都不可提現。
2. 玩家遊戲資產不能直接變成撥款。
3. 任何涉及真實金流、銀行撥款、稅務申報、外部兌換、收益暗示的功能，都必須先走高風險功能審查。

## 目前完成到哪裡

目前完成的是本地 MVP+ 原型，不是正式上線產品。

已完成：

- 前端已拆成三個入口：玩家入口、經營者中心、管理後台。
- 玩家入口已有背包、交易場行情、開包結果與不可提現邊界。
- 經營者中心已有可撥款收入、排除金額與撥款進度摘要。
- Demo Role 已收進 Demo 控制，不再作為產品主功能顯示。
- 玩家與經營者入口已有狀態式主行動按鈕，會依目前流程顯示下一步。
- 首頁已有產品邊界說明，明確區分玩家遊戲資產與經營收入撥款。
- 玩家入口已有流程進度：取得禮包、交易場、開包結果。
- 經營者中心已有撥款進度：建立結算、申請撥款、完成付款。
- 管理後台已有營運狀態摘要：營運模式、風控案件、活動與審查。
- 面向玩家的產品頁已移除內部上線缺口，只保留遊戲功能、服務規則與不可提現邊界。

## 為什麼目前還不能上線

不是只有金流未串接，而是還缺正式產品化層：

- 玩家端正式遊戲畫面尚未完成。
- 經營者端正式入駐、身份審查、合約與報稅資料尚未完成。
- 管理後台還是 MVP 營運台，不是正式客服/財務/風控工作台。
- 真實金流、銀行撥款、KYC/KYB、稅務申報尚未接入。
- 正式隱私權、服務條款、遊戲機率揭露、消費者保護文件尚未完成。
- 尚未接正式資料庫、部署環境、監控、告警與備援。
- 尚未做真實瀏覽器 E2E 測試與正式 QA 測試計畫。

## 下一步應該做什麼

優先順序：

1. 把目前 MVP 營運台整理成可展示、可理解的產品原型。
2. 補玩家端主流程頁：首頁、禮包列表、禮包詳情、交易場、開包結果。
3. 補經營者端主流程頁：收入總覽、結算紀錄、撥款申請、稅務/付款狀態。
4. 補管理端正式 IA：風控、財務、活動、審查、資料匯出分開。
5. 補正式資料模型與資料庫。
6. 補法律、稅務、金流審查後再接真實外部服務。
